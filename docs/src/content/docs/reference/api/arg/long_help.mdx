# `long_help` <LinkButton href="https://docs.rs/clap/latest/clap/struct.Arg.html#method.long_help" icon="open-book" variant="minimal"></LinkButton>

<Badge text="since 0.1.0" size="small" />

Sets the description of the argument for long help (`--help`).

# Example

### Spec

<Tabs syncKey="format">
  <TabItem label="TOML">
```toml
name = "prog"
term-width = 80
[args]
cfg = { long = "config", long-help = "The config file used by the myprog must be in JSON format with only valid keys and may not contain other nonsense that cannot be read by this program. Obviously I'm going on and on, so I'll stop now." }
```
  </TabItem>
  <TabItem label="YAML">
```yaml
name: prog
term-width: 80
args:
  cfg:
    long: config
    long-help: The config file used by the myprog must be in JSON format with only valid keys and may not contain other nonsense that cannot be read by this program. Obviously I'm going on and on, so I'll stop now.
```
  </TabItem>
  <TabItem label="JSON">
```json
{
  "name": "prog",
  "term-width": 80,
  "args": {
    "cfg": {
      "long": "config",
      "long-help": "The config file used by the myprog must be in JSON format with only valid keys and may not contain other nonsense that cannot be read by this program. Obviously I'm going on and on, so I'll stop now."
    }
  }
}
```
  </TabItem>
</Tabs>

### Input

```bash
prog --help
```

### Output

```bash
Usage: prog [OPTIONS]

Options:
      --config <cfg>
          The config file used by the myprog must be in JSON format with only
          valid keys and may not contain other nonsense that cannot be read by
          this program. Obviously I'm going on and on, so I'll stop now.

  -h, --help
          Print help (see a summary with '-h')
```
