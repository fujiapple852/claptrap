# `no_binary_name` <LinkButton href="https://docs.rs/clap/latest/clap/struct.Command.html#method.no_binary_name" icon="open-book" variant="minimal"></LinkButton>

<Badge text="since 0.1.0" size="small" />

Specifies that the parser should not assume the first argument passed is the binary name.

# Example

### Spec

<Tabs syncKey="format">
  <TabItem label="TOML">
```toml
name = "myprog"
no-binary-name = true
[args]
cmd = { num-args = "1..", help = "commands to run" }
```
  </TabItem>
  <TabItem label="YAML">
```yaml
name: myprog
no-binary-name: true
args:
  cmd:
    num-args: 1..
    help: commands to run
```
  </TabItem>
  <TabItem label="JSON">
```json
{
  "name": "myprog",
  "no-binary-name": true,
  "args": {
    "cmd": {
      "num-args": "1..",
      "help": "commands to run"
    }
  }
}
```
  </TabItem>
</Tabs>

### Input

```bash
myprog command set
```

### Output

```bash
claptrap_cmd=('myprog' 'command' 'set')
```
